Hesiod Quick Help

Note: Hesiod is a free and open-source project (FOSS) that’s still in active development. Some features may be experimental, and the application can occasionally be unstable. Please save your work often (Ctrl + Alt + S is handy), and feel free to report bugs or share feedback:

  • GitHub Issues: github.com/otto-link/Hesiod
  • Discord (preferred): discord.gg/HBgFCjdV8W

This quick guide will help you get started with Hesiod. For full details, see the documentation: hesioddoc.readthedocs.io.

Quickstart

  • Example graphs are available in the data/examples folder: they illustrate small use cases demonstrating specific node functionalities.
  • Bootstraps are available in the data/bootstraps folder: they offer minimal starting setups (Quickstart-style) to help you build your own graphs from a functional base.

Editing

Hesiod uses a non-destructive node editor: instead of permanently changing your data, you build a graph of nodes that describe how to process or transform it. Each node performs an operation, and you can rearrange, disable, or tweak them at any time without altering the original source.

  • Right-click an empty area of the graph canvas to open the node-creation menu.
  • Right-click a node to open its settings menu.
  • In the node settings, hold Ctrl to fine-tune slider parameters, or double-click a value to edit it directly.
  • Adjust graph settings (including resolution) from the More (…) icon in the editor.
  • Avoid using the layout manager to edit multiple graphs at once — it can be unstable and overly complex.
  • Use the Viewport icon to open a 3D view of the heightmap.
  • Add a Preview node and connect its input to the data you want to display.

Export

Use the export nodes in the IO/Files category to save data in your preferred format, as image files or as assets in common 3D formats (e.g., FBX, GLB).

Shortcuts

  • Ctrl + B: Add a comment
  • Ctrl + I: Import a graph
  • Ctrl + P: Automatically arrange the graph layout
  • Ctrl + Alt + S: Save a copy of the current work with a timestamp
    Handy for keeping backups if the app becomes unstable.
